Show Some Basics

Absolutely no complex action* or additional confusing configuration is required to already achieve a beautiful result with the tail.select library, whether for single or multi HTML <select> fields.

Touching code and handling some options are just required to accomplish the best feeling for your website. But don't worry, our documentation helps you at any time, and if not... Visit our GitHub Repository!

* Of course, you need to include the JS and CSS files as well as initialize the tail.select class.

Want to handle some <optgroup>s? Thinking about showing cool and additional descriptions below the respective option labels? No Problem, tail.select is there for you and does the work.

Just add the required attributes and values to your <option> elements and see how cool it can look. (This depends on the used theme, of course).

Selected options can be moved to the top of the dropdown list... Of course that's not all, you can also move them to the tail.select label (which may can destroy your website design, so test it out well), or to an own container outside.

Disabled, Selected and Hidden options can also be invisible, it's just a Configuration away.

There are 2 types of hooks you can use: The first one, called Interaction Hooks, allows you to add your own code on ANY possible action, user interaction and event made with the tail.select field.

The second one, called Plugin Hook, allows you to extend and improve the complete tail.select environment. It is called on any instance and internal interaction to give you all available possibilities.
