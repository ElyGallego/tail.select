Public Options

The current version of our tail.select library contains 41 settings. All of these can be passed directly as second parameter on the main tail.select() constructor, but keep in mind that a few settings (such as disabled and multiple) gets overwritten by the source <select> HTML element itself, but only on the first real initialization*. However, you still can change them at any time later using the .set() method.

Explanation: The first real initialization* happens, when the constructor is called on <select> elements, which aren't already instantiated by a tail.select() contructor before.

height Added in 0.2.0 Updated in 0.6.0

This option sets the max-height of the dropdown container. Just pass an integer or a string with any number and the unit type of choice, to calculate the height into a pixel unit. You can also define the height based on the maximum amount of options to be displayed, by passing a string starting with a colon and followed by the number of options. (ex.: ":10").

It's not recommended, but you also can set this option to null which will always use the maximum required and available space as max-height value, depending on openAbove.

Good to know: The passed height will always contain the height of the search field too, if enabled, unless you're using the new ":<number>" syntax.

items Added in 0.3.0 Updated in 0.6.0

This option allows you to add new items, which gets added to the tail.select instance as well as to the source <select> HTML field. You need to pass your items as key: value paired Object, while the value can either be just the innerText value or another object containing at least a value property with the respective innerText string.

Simple Example

The simple example doesn't allow any additional attributes, so you can't pre-select, disable or hide anything. They just gets shown normally and added as ungrouped items. 


tail.select(".select", {
    items: {
        option1: "Option 1",
        option2: "Option 2",
        option3: "Option 3"
    }
});
Advanced Example

The more advanced way allows you to define each state and attribute of the new <option> element, as well as the optgroup and position where the new item should be added. The general known keys are:

value
The value is required and used as title (innerText) of the respective <option>. Don't be confused with the value attribute used on the <option>-element, because this will be the assigned key property (so option1 on the example below).
optgroup (or group)
The <optgroup> label, where the new <option> should be added to. Skip this property or just use # to add the new <option> as ungrouped item. The <optgroup> get created automatically, when no <optgroup> with this label could be found.
position
The position number where the new <option> should be added. Use -1 to add the new <option> on the end, or use the respective number for example 0 for the first position.
selected
The boolean value of the desired selected state.
disabled
The boolean value of the desired disabled state.
hidden
The boolean value of the desired hidden state.
description
The content for the data-description option attribute.
data-*
You can also add your own custom data-* attributes to the new <option> element.

tail.select(".select", {
    items: {
        option1: {
            value:          "The Value Property is required!",
            group:          "My Optgroup",
            position:       -1,
            selected:       false,
            disabled:       false,
            hidden:         false,
            description:    "My Description Text"
        }
    }
});

linguisticRules Added in 0.5.9 Updated in 0.6.0

This option contains an object with all available linguistic rules, which gets applied on the search function to changes language-depending characters into English ones. The English equivalent character is set as a property and the language-specific character as a value, in German it would look like this { 'a': 'ä', 'o': 'ö', 'u': 'ü', 'ss': 'ß' }.

Each tail.select language file adds his own lingustic rules to this object. Check out the Translation page to see which characters are already included. You can also disable the use of the lingustic rules, by adding 'strict' to the searchConfig option.

Example: In German you are able to write Street as "Straße" (using an 'Eszett') and 'Strasse' (using two 'ss'), by applying the lingustic rules you're able to use both spellings to find the same results.

placeholder Added in 0.1.0 Updated in 0.6.0

This option sets a custom placeholder on the respective tail.select instances, which can be directly passed as string or as callback function. Keep in mind, that this option gets overwritten by the data-placeholder attribute of the source <select> field on the first real initialization.

The callback function gets triggered every time the API Method .updateLabel() is called. It gets passed the respective tail.select instance as this argument as well as the following four parameters. This function MUST always return a string back, if you don't want to change anything just return the first passed parameter.

Callback Arguments

string
The complete parsed label string, which would be used per default.
label
The unparsed label locale id.
count
The unparsed count string or null if no count should be shown.
replace
The replacement object, which contains 3 properties:
  • '[0]': <value> - The number of selected options.
  • '[1]': <value> - The multiLimit option value or &infin; (∞).
  • '[2]': <value> - The number of still selectable options.
  • Changelog

  • null

placeholderCount New Added in 0.6.0

This option sets the count / limit string left to the placeholder (or right of it if rtl is enabled). You can either use a string containing one of the following words, or a callback function. Keep in mind that this option works only on multiple <select> fields.

Possible Strings

count-up
Shows the number of selected options.
count-down
Shows the number of options that still can be selected depending on the multiLimit option. If this option is set to Infinity the number of selectable, not-selected options gets returned.
limit
Shows the static multiLimit number or ∞ if set to Infinity.
both
Combines 'count-up' / 'limit', including the slash.

Callback

The callback function gets triggered every time the API Method .updateLabel() is called. It gets passed the respective tail.select instance as this argument as well as the following two parameters. You can return one of the property values (as shown on the replace parameter), any kind of string or just null to skip the count part at all.

Callback Arguments

label
The unparsed label locale id.
replace
The replacement object, which contains 3 properties:
  • '[0]': <value> - The number of selected options.
  • '[1]': <value> - The multiLimit option value or &infin; (∞).
  • '[2]': <value> - The number of still selectable options.

Good to know: This new option replaces the multiShowCount and multiShowLimit options.

searchConfig Added in 0.5.13

This option allows you to highly configure or completely take over the search functionality and has been written by @Calak (Many Thanks!). To configure the search system, just pass an Array with one ore more of the following keys (you can also pass just one single key as string too). To take over the search completely, just pass a function.

any
Search in all available attributes and in the <option>s value and innerText too.
attributes
Search in all available attributes.
value
Search in the <option>s value and innerText.
text
Perform a textual search, which removes the most special characters.
regex
Perform a regular espression pattern search.
case
Force a case-sensitive search.
strict
Prevent applying the linguistic rules, as defined in the linguisticRules option.
word
Force the search value to be a whole / outstanding word.

titleOverflow New EXPERIMENTAL Added in 0.6.0

This option allows you to set the behaviour on too long option titles within the tail.select dropdown field. You can either break up the text onto multiple lines (using 'break'), clip them and show an '…' on the end of it (using 'clip') or let them scroll to show the complete text, if the cursor points over it (using 'scroll').

Keep in mind: The available option 'scroll' is EXPERIMENTAL and doesn't may work as expected!

Available Values

break
Breaks up the text onto multiple lines.
clip
Clips the text and adds an '…' on the end.
scroll
Clips the text and adds an '…' on the end, but scrolls if the cursor is over it.
