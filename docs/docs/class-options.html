Options Class

The Options class handles the single <option> elements inside the source <select> fields. It offers different methods to add, get, modify, remove and walk through the single items offering a few alias methods for an easier usage.

Main Constructor Added in 0.3.0 Updated in 0.6.0

The main construtor is called inside the Select.init() method and just requires the respective tail.select instance as first and only parameter. It prepares the single <option> elements with the assurance that each one has a value attribute and it forces the deselectability if the deselect option is enabled on single <select> fields.

.create() New Added in 0.6.0

This helper methods easily creates a new valid <option> element containing at least an inner text as well as a value attribute.

Arguments

value StringRequired
The value attribute used on the new HTMLOptionElement object.
text StringRequired
The innerText value used on the new HTMLOptionElement object.
data Object
An addiitonal object containing all attributes and states you want to set on the new HTMLOptionElement object. The object keys 'group', 'optgroup' and 'position' (as used on the .add() method) are skipped, and you can use 'description' (instead of 'data-description').
Object
This method ALWAYS returns the HTMLElementObject, created with the respective parameters.

.get() Added in 0.3.0 Updated in 0.6.0

This method allows you to select all available, or just a bunch of specific options depending on the parameter you've passed. In Fact it builds up a selector string and uses them inside (or including) the source <select> element.

Arguments

value String | null
You can either pass the value of a specific <option> element or the position number (which can also be negative to start counting from the end). Keep this argument on null to disable this selection restriction. Passing an Object will cancel this method and returns itself back again.
group String | null
You can either pass the <optgroup> label value (or use # for the ungrouped group), to restrict the selection to this single group or you keep this on null to disable this selection restriction.
checked Boolean | null
Pass true to explicit get selected options, false to explicit exclude them or just null to disable this selection restriction.
disabled Boolean | null
Pass true to explicit get disabled options, false to explicit exclude them or just null to disable this selection restriction.
hidden Boolean | null
Pass true to explicit get hidden options, false to explicit exclude them or just null to disable this selection restriction.
Object
This method returns an Array with all selected <option> elements, selected using the passed arguments or the first parameter itself, if it is an Object.

.getGroups() New Added in 0.6.0

This method allows you to select all <optgroup> HTMLOptGroupElements or just the respective label attributes (depending on the first parameter), inside the source <select> field.

Arguments

objects Boolean
Pass true to explicit return the <optgroup> elements inside a NodeList Object, pass false or just skip this parameter to just return the label values within an Array.
Object | Array
All available <optgroup> elements as NodeList or just the label strings inside an Array.

.count() New Added in 0.6.0

This method allows you to count the available <option> elements within the source <select> field. It allows using the same arguments structure, with Exception of the first on, as used on the .get() method.

group String | null
You can either pass the <optgroup> label value (or use # for the ungrouped group), to restrict the selection to this single group or you keep this on null to disable this selection restriction.
checked Boolean | null
Pass true to explicit get selected options, false to explicit exclude them or just null to disable this selection restriction.
disabled Boolean | null
Pass true to explicit get disabled options, false to explicit exclude them or just null to disable this selection restriction.
hidden Boolean | null
Pass true to explicit get hidden options, false to explicit exclude them or just null to disable this selection restriction.
Number
The number of <option> elements inside the source <select> field, depending on the arguments you passed. If you pass nothing, it returns the number of all currently available elements.

.set() Added in 0.3.0 Updated in 0.6.0

This method allows you to set a new <option> element within the source <select> field. This is also used on the .add() method.

Arguments

item StringRequired
You can pass either a single HTMLElement object or multiple ones inside an Array.
group StringRequired
The group label value (use # for the ungrouped group), where the new <option> elements should be added. If the label doesn't exist, it will create and add a new one.
position Object
The position number, where the new <option> element(s) should be added. The default value is -1, which sets the option always on the end of the list.
reload Object
This argument soft .reload()s the dropdown field after all passed <option> elements (see first parameter) has been set. Use false to disable this behaviour.
Object
This method always returns the Options instance object itself.

.add() Added in 0.3.0 Updated in 0.6.0

This method allows you to add a bunch of new <option> elements, within the source <select> field, using a well-formatted Object as first argument. This object gets parsed, created with the .create() method and gets finally added using .set().

Arguments

data ObjectRequired
The data object, containing all items you want to add. Check out the items option to see how this Object MUST be.
group StringRequired
The group label value (use # for the ungrouped group), where the new items should be added. This argument doesn't has any effect, if the first argument already contains the single optgroup labels.
reload Object
This argument soft .reload()s the dropdown field after all passed <option> elements (see first parameter) has been added. Use false to disable this behaviour.
Object
This method always returns the Options instance object itself.

.handle() Added in 0.3.0 Updated in 0.6.0

This method changes all states, as defined in the second parameter, on all elements of the first parameter. It also calls the Select own .update() method to communicate all changes on all items, unless you prevent this call by passing true as third parameter.

Arguments

items String | Array | Object | nullRequired
Pass an HTMLElementOption, a NodeList or a HTMLCollection containing all <opton>s which should be changed. You can also pass a single string or an Array, which gets applied to the .get() method. You can also pass null which will change all available <option> elements at once.
states ObjectRequired
A state object, where the state name you want to change is the key pointing to the new state value (using a boolean value) or null to toggle the respective state. A state object can contain multiple states at once, for example: { selected: null, disabled: true } will toggle the selected state and disable the option regardless if it is selected or not.
prevent Boolean
Using true prevents the call of the parent .update() method, otherwise it gets called with the changed items Array as described on the returning value below.
force Boolean
Using true will force the change (which effects the 'selected' state only), even if it shouldn't be possible. For example: Set this to true to select another <option> even if the multiLimit is already reached.
Array
An Array, which elements are another Arrays consisting of the source <option> element and an object containing all done state changes. For example: [ [<option>, { selected: true }], ... ].

.find() Added in 0.3.0 Updated in 0.6.0

This method walks through all available <option> fields and applies the search term (first parameter). Keep in mind that selected, disabled or hidden <option> fields gets skipped if the respective option hideSelected, hideDisabled or hideHidden is set to null.

Arguments

search String Required
The search term, which should be applied.
config String | Function | null
The search configuration, which can either be a function or an array with one or more configuration strings. Check out the searchConfig option for more informations.
order String | Function | null
The ordering instruction for the search results. You can either use 'ASC' or 'DESC' as string, or pass an custom order callback function. Pass nothing or null to keep the current order. Check out the sortSearch option for more informations.
Array
An Array with all resulting <option> elements.

.finder() Added in 0.3.0 Updated in 0.6.0

The finder method calls the .find() method and returns the result either as Pseudo-Generator or as real Generator, depending on which build you're using. Keep in mind that selected, disabled or hidden <option> elements gets skipped if the respective option hideSelected, hideDisabled or hideHidden is set to null.

ECMAScript 5

The Pseudo-Generator, as used on the ECMAscript 5 build, returns each single item one per one and false if not item is left. Of course, you can break the loop before reaching the end, which requires you to call this method again by passing reset as this argument.

ECMAScript 2015

The real Generator, as used on the ECMAScript 2015 build, yields (MDN) each single item until it reaches the end or you break the loop on your own. You don't need to reset them, since real generators doesn't need to be reseted.

Arguments

search String Required
Check out the first argument on the .find() method.
config String | Function | null
Check out the second argument on the .find() method.
order String | Function | null
Check out the third argument on the .find() method.
Object | False | *Generator
Returns the single <option> objects one by one or false if no item is left on the ECMAScript 5 build, or a real Generator on the ECMAScript 2015 build.

.walker() Added in 0.3.0 Updated in 0.6.0

The walker method returns each single <option> element either as Pseudo-Generator or as real Generator, depending on which build you're using. Keep in mind that selected, disabled or hidden <option> elements gets skipped if the respective option hideSelected, hideDisabled or hideHidden is set to null.

ECMAScript 5

The Pseudo-Generator, as used on the ECMAscript 5 build, returns each single item one per one and false if not item is left. Of course, you can break the loop before reaching the end, which requires you to reset it on method again by passing reset as this argument.

ECMAScript 2015

The real Generator, as used on the ECMAScript 2015 build, yields (MDN) each single item until it reaches the end or you break the loop on your own. You don't need to reset them, since real generators doesn't need to be reseted.

Arguments

orderg String | Function | null
The ordering instruction for the <optgroup> labels. You can either use 'ASC' or 'DESC' as string, or pass an custom order callback function. Pass nothing or null to keep the current order. Check out the sortGroups option for more informations.
orderi String | Function | null
The ordering instruction for the <option> objects. You can either use 'ASC' or 'DESC' as string, or pass an custom order callback function. Pass nothing or null to keep the current order. Check out the sortItems option for more informations.
Object | False | *Generator
Returns the single <option> objects one by one or false if no item is left on the ECMAScript 5 build, or a real Generator on the ECMAScript 2015 build.

Removed Methods

The following methods has been removed or replaced.

all() Added in 0.5.0 Removed in 0.6.0

This method allowed to loop and handle all available <option> fields at once. Since Version 0.6.0 you can achieve the same by calling the .handle() method, or one of his alias methods, using null as first parameter.

init() Added in 0.3.0 Removed in 0.5.0

This method initialized the Options instance, which is now completely done by the constructor and doesn't require an own method anymore.

is() Added in 0.3.0 Removed in 0.6.0

This method has been removed to keep the source a few lines smaller. Since Version 0.6.0 the tail.select script also works directly with the single <option> elements, instead of storing and handling them on our own. So please use the source code as well instead!

invert() Added in 0.3.0 Removed in 0.6.0

This method inverted the passed state on all (passed) items. Since Version 0.6.0 you can achieve the same by calling the .handle() or .toggle() method, using null as first parameter as well as on the respective state.

.move() Added in 0.5.0 Removed in 0.6.0

This method has been removed to keep the source a few lines smaller. Since Version 0.6.0 the tail.select script also works directly with the single <option> elements, instead of storing and handling them on our own. So please use the source code as well instead!

.remove() Added in 0.3.0 Removed in 0.6.0

This method has been removed to keep the source a few lines smaller. Since Version 0.6.0 the tail.select script also works directly with the single <option> elements, instead of storing and handling them on our own. So please use the source code as well instead!

This method has been removed to keep the source a few lines smaller. Since Version 0.6.0 the tail.select script also works directly with the single <option> elements, instead of storing and handling them on our own. So please use the source code as well instead!
