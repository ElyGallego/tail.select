Select Class

The select class is the main handler which initiates and takes care about the complete tail.select script. Except the handling of the single <option> elements inside the source <select> field, because this gets processed and performed by the Options class.

Main Constructor

The main constructor does the first real initialization, which consists of adding a unique ID, sets the options multiple, disabled and placeholder (using the values of the source <select> field) and stores each single instance on the static select.inst variable. This means, that you can use the constructor on already initialized <select> fields without building up a new instance.

Arguments

element String | Array | ObjectRequired
The selector string, which gets applied on the documents .querySelectorAll() function, for all <select> fields you want to tail.selectify. You can also directly pass the single HTMLElement or any array(-like object), such as an Array, a NodeList or a HTMLCollection, containing all the <select> fields you want.
config Object
An object containing all key: value paired options you want to change from the default value. Each object gets cloned, so you can init multiple <select> fields with the same object, but change them individually later. Check out the Options page to see all available options.
options Object
You can pass a custom or modified Options class, which is used to handle the single <option> elements on all source <select> fields defined in the first parameter. Of course, your custom handler class MUST contain all core method names.
Void | Object | Array
Passing a single HTMLElement object as first parameter will just return the single tail.select object instance on success or void 0 alias undefined on error. (Note: If you're using the new keyword on the select constructor, which isn't recommended, an empty Object will return on error instead.)

Otherwise, if you passing just a selector string or an array(-like object), it will return an Array with all valid and created tail.select object instances or an empty array if no valid <select> field was available. 

tail.select(
    "select",                   // The Selector / Element / Array(-Like Object)
    { },                        // (Optional) Your Custom Configurations
    tail.select.options         // (Optional) Your Custom Options Object
);

Keep in mind: The tail.select constructor above works only on the ECMAScript 5 build using the window-global hooked method. Check out the Use Sections on the Basics Page to learn more about that.

jQuery Binding

The ECMAScript 5 build also contains a custom binding for the jQuery library. The tail.select constructor is bind to tailselect (without the dot) and returns ALWAYS an Array containing all instance objects or an empty Array if no valid <select> field is available. 


jQuery("select").tailselect(
    { },                        // (Optional) Your Custom Configurations
    tail.select.options         // (Optional) Your Custom Options Object
);

MooTools Binding

Next to jQuery you can also use the MooTools JavaScript library with the ECMAScript 5 build. The tailselect method is also implemented in both core objects, Element and Elements, to replicate the original returning value.

This means, if you're using the single $ selector (which is an alias for document.id) you'll receive either the single tail.select instance object or just void 0 alias undefined. Using the twice $$ selector function will always return an Array containing all valid instance objects, or an empty Array if no valid <select> field is available. 


$('my-select').tailselect(
    { },                        // (Optional) Your Custom Configurations
    tail.select.options         // (Optional) Your Custom Options Object
);

$$('select').tailselect(
    { },                        // (Optional) Your Custom Configurations
    tail.select.options         // (Optional) Your Custom Options Object
);

Internal Methods

The following methods are marked as internal, so they shouldn't replaced by yourself. You can use Hooks and Filters, as described on the Callbacks page, to inject your own code or to completely take over a few of these.

._e() Added in 0.5.8 Updated in 0.6.0

This method handles the complete translation and localization of each available string. It allows to pass the string id as first, and a replacement object as second parameter.

Arguments

string StringRequired
The string ID as it is stored in the .__ locale object variable.
replace Object
A key: value paired object, where the key is the substring to be replaced with value.
String
The localized, and with the replace object parsed, string on success or the passed first parameter on failure (such if the first parameter couldn't be found on the main .__ locale object).

._cls() New Added in 0.6.0

This method generates the class names required for the item element inside the main tail.select dropdown field. Custom class names can also directly passed as second parameter, using a space-separated string.

Arguments

item ObjectRequired
The source <option> HTMLElement object, which is used to generate the class names.
string String
The custom class names as space-separated string, which gets prepended to the required ones.
String
The string containing all required and custom class names.

.init() Added in 0.3.0 Updated in 0.6.0

This method initialize the tail.select instance on the first real initialization as well as everytime the .reload() method is called with passing false as first parameter.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

.build() New Added in 0.6.0

This method builds up all the elements required, and optionally used, by the tail.select script and should get called only by the internal .init() method itself.

Arguments

self ObjectRequired
The first parameter is the instance object itself, which is used for the event listeners only.
Object
This method always returns the Select instance object itself.

.bind() Added in 0.3.0 Updated in 0.6.0

This method attach, detach and is served for the event listeners which are set on the main document object as well as on the source select field directly if the sourceBind option is enabled. The listener callback is stored on the .listen variable.

Arguments

arg Object | nullRequired
The first parameter can be different values. If it is the Select instance itself, it will attach the respective event listeners, if null is passed it will remove the attached listeners and if the Event object is present it will handle the respective events.
any
The returning value varies depending of the passed parameter.

.calculate() New Added in 0.6.0

This method calculates the height of the tail.select dropdown field, depending on the used height option and if a .dropdown-inner container is available.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

API Methods

The API methods handles and controls the important things of the tail.select script. You can freely call them yourself, for example building up a more complex environment or a plugin. Just use them with care, because it may break up everything if you do something wrong.

.trigger() Added in 0.4.0 Updated in 0.6.0

This method does the most magical on the complete tail.select script, because it triggers the events, applies the filters and handles all attached hooks. It is the main interface between you, your script, all available plugins and the core system itself.

Arguments

type StringRequired
The callback type, which should be triggered. Use event, filter or hook. Keep in mind that each single event gets also triggered, with a tail::-prefixed name, on the main tail.select container.
name StringRequired
The callback name, for example change or open. Check out the Callbacks page to see all available events, filters and hooks.
args Array
The arguments, which should be passed to the attached callback functions. This Array also gets added to the Event's own detail object, but only when the callback type is set to event of course.
Boolean | Array
The returning value varies depending on the first parameter. If you trigger a hook and event it will always return a boolean value, which is only important for hooks. Triggered filters will always return the filtered args Array you've passed as third parameter.

.query() Added in 0.5.0 Updated in 0.6.0

This method starts building up the tail.select dropdown field using the method, which is passed as first parameter, called with the arguments passed as second one. The passed method MUST be a pseudo-generator (which returns all items one per one and False on the end) by using the ECMAScript 5 build and a real-generator on the ECMAScript 2015 (ES6) build.

Arguments

method String | FunctionRequired
The method, which should be used for the query. This can be a string representation of an method, which is available in the Options class or a direct callback function.
args Array
The arguments, which gets passed to the defined query method.
Object
This method always returns the Select instance object itself.

.render() New Added in 0.6.0

This method offers all available rendering functions, as used in the query() method, to build up the tail.select dropdown field. The core method contains rendering functions for the group and list items as well as for empty select fields.

Arguments

type String | FunctionRequired
The render type, which should be used. The core method allows to pass 'empty', 'group' and 'item'.
data any
Some additional data, which is required for the rendering. It gets passed as first parameter on the respective render function.
query Array | null
The last query Array, as described on the _last_query variable. It gets passed as second parameter on the respective render function.
Object | False | null
The returning value should always be the respective HTMLElement object, which will be used. Within the loop, as used inside the .query() method, it is also allowed to return null, which skips the current rendered group or item, or false which breaks the loop completely.

.update() Added in 0.5.0 Updated in 0.6.0

This method updates the tail.select script and triggers all required callbacks, when a state of an item (such as disabled, selected or hidden) gets changed. It also calls the .updateCSV() as well as the .updateLabel() methods.

Arguments

items Array
An Array, which elements are another Arrays consisting of the source <option> element and an object containing all done changes. [[<option>, { selected: true }], ...]. The update method will be completely skipped if this argument is either undefined or an empty Array.
trigger Boolean
A boolean value, which decides if the event callbacks should be triggered (true) or not (false).
force Boolean
A boolean value, which indicated if the changes has been forced or not. A force change, for example, is when more options has been selected as the multiLimit option allows.
Object
This method always returns the Select instance object itself.

.updateCSV() Added in 0.5.0 Updated in 0.6.0

This method just updates the CSV <input /> value, if the csvOutput option is enabled. It uses the filtered output of the .value() method and adds them directly.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

.updateLabel() Added in 0.5.0 Updated in 0.6.0

This method updates the label and counter, as used on the tail.select label element, respecting also the used placeholder and placeholderCount options.

Arguments

label String
You can pass a custom label string, but keep in mind that this may gets overwritten if the placeholder option is a callback function.
Object
This method always returns the Select instance object itself.

Public Methods

Public methods can freely used to command, configure or work with the tail.select instance. Just use them inside your scripts, however, keep the partially required parameters correctly.

.open() Added in 0.3.0 Updated in 0.6.0

This method opens the tail.select dropdown field, regardless of the state except it is already open. It also calls the .calculate() method to redefine the dropdown's height and focus on the search input field if both search and searchFocus are enabled.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

.close() Added in 0.3.0 Updated in 0.6.0

This method closes the tail.select dropdown field, regardless of the state except it is already closed. It also removes the height properties, which has been set by .calculate(), again.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

.toggle() Added in 0.3.0 Updated in 0.6.0

This method just toggles the tail.select dropdown field by calling .open() or .close(), depending if the dropdown field is opened or closed.

Arguments

This method doesn't allow any arguments.
Object
This method always returns the Select instance object itself.

.reload() Added in 0.3.0 Updated in 0.6.0

This method allows to reload the tail.select instance, which may must be done if you change something on the source <select> field or you change any important option using .set(). This method also gets called by a few Options class methods.

Arguments

soft Boolean
Passing true will just re-call the .query() method with the last used arguments. Otherwise, by using false, the complete instance gets removed and rebuild calling the methods .remove(false) and .init().
Object
This method always returns the Select instance object itself.

.remove() Added in 0.3.0 Updated in 0.6.0

This method removes the complete tail.select instance as well as all changes made on the source <select> field as far as possible. You can just prevent the deletion of later-added options, for example using the items option, by passing true as first parameter.

Arguments

keep Boolean
This argument allows you to keep (true) options and optgroups which has been added later, using the items option or the .set() method on the Options class. The default is false, which means that all added <option> elements gets removed too.
Object
This method always returns the Select instance object itself.

.get() New Added in 0.6.0

This method allows you to get the value of an option set using the main Select Constructor or at any time later using the .set() method. Keep in Mind: The default option values are NO LONGER stored in the script files itself, so if you didn't set the option on your own, you won't receive the real value (unless you pass them as second parameter).

Arguments

key StringRequired
The option key as string.
def any
The default value, if the option key isn't set.
any
This method returns the respective value of the option key you passed or the second parameter if the option key doesn't exist. If you don't pass a second parameter void 0 will return.

.set() New Added in 0.6.0

This method allows you to set and change the value of an option. Keep in mind, that you need to .reload() the tail.select instance yourself if you set or change any option which influence the script, the main elements or the source <select> field.

Arguments

key String | ObjectRequired
The respective option key you want to change as string, you can also pass an object containing multiple key: value paired properties to change multiple options at once.
value any
The new value, which should be set. This works only if the first parameter is a string, of course.
Object
This method always returns the Select instance object itself.

.enable() Added in 0.5.0 Updated in 0.6.0

This method enables the tail.select instance as well as the source <select> field, and soft-reloads the instance using the .reload() method.

Arguments

reload Boolean
A boolean value, which decides if the .reload() method should be called (true) or not (false).
Object
This method always returns the Select instance object itself.

.disable() Added in 0.5.0 Updated in 0.6.0

This method disabled the tail.select instance as well as the source <select> field, and soft-reloads the instance using the .reload() method.

Arguments

reload Boolean
A boolean value, which decides if the .reload() method should be called (true) or not (false).
Object
This method always returns the Select instance object itself.

.on() Added in 0.4.0 Updated in 0.6.0

This method allows you to add your own event, filter and hook callbacks to the instance. You can also use the on option to attach your callbacks directly to the first initialization. Check out the Callbacks page to see all available callbacks.

Arguments

name String | ArrayRequired
The callback name as string, you can also pass multiple callback names as Array which all gets attached with the same callback function, passed as second parameter.
func FunctionRequired
The callback function, which should get called, on all passed callback names defined on the first parameter.
Object
This method always returns the Select instance object itself.

.value() Added in 0.5.13 Updated in 0.6.0

This method allows you to receive the current selected values in different formats.

Arguments

format String
The format of how the value should be returned ('auto' is the default).
csv Returns all values as string, separated using the csvSeparator option
nodes Returns all option elements as NodeList object.
array Returns all values inside an array.
auto Use 'csv' on single, and 'array' on multiple <select> fields.
String | Array | Object
This method returns the selected option (values) as described above.

Variables

The Select class offers the following variables, which we strongly discourage changing it, unless you really know what you're doing and what may happens if you're doing it wrong. However, you can still use them as readonlys for your own script, just handle them with care.

.__

This variable contains the translation string object of the current set locale option. It is used mainly on the ._e() method.

._select

This variable contains the HTMLElement object of the main tail.select container and can be used, for example, to catch triggered events using the .addEventListener() function.

._label

This variable contains the main HTMLElement object for the label, which prints the placeholder and placeholderCount options as well as the arrow toggle button.

._dropdown

This variable contains the main HTMLElement object for the dropdown field, which contains the search container, if the search option is enabled, as well as the dropdown-inner container which contains all the <ul> and <li> list elements.

._search

This variable contains the main HTMLElement object for the serch field, which itself contains the respective <input /> input field only.

._csv

This variable contains the hidden <input /> field HTMLElement object as used for the csvOutput option, even if this option is disabled!

._last_query

This variable contains the last used arguments of the .query() method, which consists of the used query method as first and the query arguments as array as second parameter.

.id

This variable contains the unique tail.select id, which has been counted up on the first real initialization. It is used on the id attribute of the main container and linked on the source <select> element.

.e

This variable contains the source <select> HTMLElement object.

.con

This variable contains the configuration object of all options, which has been set using the tail.select() constructor or the .set() method. At least this object contains ALWAYS the 'multiple', 'disabled', 'required' and 'placeholder' keys (even if the last one is maybe set to undefined).

.listen

This variable contains just the event listener callback function, set in the .init() method and used only within the .bind() method. The function also just returns .bind() itself, which this argument has been bind to the respective instance itself.

.callbacks

This variable contains all callback functions for the events, filters and hooks, which has been set by the .on() method or the on option.

.options

This variable contains the Options class, respectivly it's instance after the .init() method has been applied the first time.

.plugins

This variable contains the Plugins class instance.

Removed Methods

The following methods has been removed or replaced.

.callback() Added in 0.3.0 Removed in 0.6.0

This method was used prior version 0.6.0 for the communication between the Options class and the Select class. The existing method .update() is now used instead.

.calc() Added in 0.5.0 Removed in 0.6.0

This method was used prior version 0.6.0 to calculate the height of the dropdown field. For a clear naming it was renamed into .calculate().

.cbGroup() Added in 0.4.0 Removed in 0.6.0

This method was used prior version 0.6.0 to generate / render the single dropdown item groups. The new .render() method, using "group" as first parameter, is now used instead.

.cbItem() Added in 0.4.0 Removed in 0.6.0

This method was used prior version 0.6.0 to generate / render the single dropdown items. The new .render() method, using "item" as first parameter, is now used instead.

.choose() Added in 0.3.0 Removed in 0.5.0

This method was used prior version 0.5.0 as alias for the Options class to handle the single <option>s on the source <select> field.

.config() Added in 0.4.0 Removed in 0.6.0

This method was used prior version 0.6.0 to get or set (parts of) the configuration object. It has been replaced by the new .get() and .set() methods.

.setLabel() Added in 0.3.0 Removed in 0.5.0

This method was used prior version 0.5.0 to handle and set the current placeholder label on the main tail.select label element. It has been replaced with the .updateLabel() method.

.setContainer() Added in 0.3.0 Removed in 0.5.0

This method was used prior version 0.5.0 as the multiContainer option and functions were still part of the core system. This behaviour was replaced by the .updateContainer() method, which itself has been removed in version 0.6.0 and is now available on the select-movement Plugin.

.setCSVInput() Added in 0.4.0 Removed in 0.5.0

This method was used prior version 0.5.0 to handle and set the CSV <input /> value, as used by the csvOutput option. It has been replaced with the .updateCSV() method.

.setCounter() Added in 0.3.0 Removed in 0.5.0

This method was used prior version 0.5.0 to calculate and print the counter element inside the main tail.select label container. This functionallity is still available, but has been add to the .updateLabel() method.

.updateContainer() Added in 0.5.0 Removed in 0.6.0

This method as used prior version 0.6.0 as the multiContainer option and functions were still a part of the core system. The complete behaviour has now been moved to the new select-movement Plugin.

.updatePin() Added in 0.5.0 Removed in 0.6.0

This method as used prior version 0.6.0 as the multiPinSelected option and functions were still a part of the core system. The complete behaviour has now been moved to the new select-movement Plugin.
