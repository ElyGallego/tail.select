Callbacks

The tail.select environment offers you 3 different types of callbacks, where you can place your own code, interact with a specific part of the tail.select script or also completely take-over an entire workflow of the method itself.

You can add each callback type using the on option on the main tail.select() constructor or at any time after the initialization using the .on() method on the respective instance.

Events

"Something happened, and you may need to know it!"

Events gets triggered everytime, when some (user) interaction happens, such as the dropdown field gets opened, the bunch of options gets rendered in the tail.select dropdown field or an item gets selected, disabled or hidden.

Events gets also attached and triggered on the main tail.select container, which means that you can catch them using .addEventListener() next to the own .on() method.

Good to Know: Changing a state on an item gets also triggered on the source <select> field!

Filters

"Something has been created, and you may want to change it!"

Filters gets triggered everytime, when something important has been created or initialized, such as a rendered dropdown item or the method and arguments of a single .query(). Attached callback functions will always gets passed the respective arguments, which you're able to change and MUST return the arguments again (even if you don't changed them).

Good to Know: It's highly important to ALWAYS return ALL arguments in the same order. Otherwise the script will break!

Hooks

"Something is or has been done, and you may want to intervene!"

Hooks gets triggered everytime on the top and the bottom of important Select() methods, and ALWAYS contains a :before or :after on the callback name. Both types gets ALWAYS passed the arguments with which the respective method itself was called too.

The :before hook allows you to prepend your own code or to completely take-over the entire workflow of the respective method, depending if you return true (prepend) or false (take-over) on your callback function. The :after hook just allows you to append your own code on the bottom of the method.

Good to Know: Just one of the attached callbacks needs to return false to skip the entire workflow!
