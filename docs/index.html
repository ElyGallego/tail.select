Some Basics

Embed and Initialize

Absolutely no complex action* or additional confusing configuration is required to already achieve a beautiful result with the tail.select library.

Touching code and handling some options are just required to accomplish the best experience for your website. But don't worry, our extensive documentation helps you out at any time. And if not... Visit and Ask us directly on our GitHub Repository!

* Of course, you need to include the JS and CSS files as well as initialize the tail.select class.

Show a bit more

Want to handle some <optgroup>s, maybe with an own custom placeholder too? Thinking about showing additional and explaining descriptions below the respective <option> labels? No Problem! You just need to adapt your HTML <option>s structure.

The multi selection buttons are added per default, but the sticky group behaviour, as shown on the second example, must be added manually.

Search & Limitations

Easify the Selection

Sometimes it would be great to search through the bunch of options you're offering and sometimes it would be better to limit the options the user can select. Both can be handled with ease using a few settings.

Tip: Using thousands of options may slow down your website! Don't show them all at once, achieve more speed and flexability with our AJAX Plugin.

Use some Magic

Allow the deselection of single <select> fields, use a hidden CSV output field, change the text direction, use custom class names, hide what you don't want to see, handle and hook the source <select> field, control and deal with the dropdown field and open / close actions.

The base system already offers so many functions, that the most needs should be covered. For the rest there are lots of hooks, and maybe the right plugin too.

It's all about the hooks

Everything you need to go deeply to the single processes are the everywhere involved hooks. They are so important that there are 3 types of them:

The Events gets triggered if something happend you may need to know, the Hooks gets called if something is or has been done and you may want to intervene, the Filters gets applied if something is or has been created you may want to change.
