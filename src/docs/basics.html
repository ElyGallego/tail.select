Installation & Usage

The tail.select script is available as global hooked and AMD-ready ECMAScript 5 version, which is compatible with the Internet Explorer 10 and above, as well as ECMAScript 2015 module (also known as ES6) which requires a more modern browser (but should also be compatible with the EdgeHTML and EdgeChromium browsers).

Explaning the Versions

ECMAScript 5

The ECMAScript 5 build should ALWAYS be used, when building up a website or a web-based platform, also if you don't care about Internet Explorer or other Older-Browser at all. Wherever possible, "more modern" JavaScript language features are tried, containing always a fallback solution if they aren't available. It's perfect to offer your website for many people as possible.

ECMAScript 2015 (ES6)

The ECMAScript 2015 build should ONLY be used, when building up a ES6 module-based website or web application, which was only developed for modern browsers anyway. It just uses "modern" JavaScript language features, which has been marked as stable and integrated in the well-known browser-engines for "enough" time.

Get tail.select

You can download the latest stable version on GitHubs Release page at any time, or by using the above ZIP and TAR buttons on the main navigation. Keep in mind, that it isn't recommended to download the latest master branch, because it may contains changes and updates which aren't fully tested and may gets rejected later.

Using a Package Manager

You can also download the latest release using the package managers npm, yarn or bower. Don't install them globally, because this wouldn't work, even if the ECMAScript 5 build contains a node.js module loader interface. The tail.select script is not compatible with node.js! 

npm install tail.select --save
yarn add tail.select --save
bower install tail.select --save

Using a CDN

A CDN may speed up your website and is a helpy little possibility to easily add the tail.select script to online Editors and IDEs like jsFiddle or CodePen.io. Just visit the following URLs and navigate to the respective files you want to include: 

https://cdn.jsdelivr.net/npm/tail.select@latest/
https://unpkg.com/tail.select/

Use tail.select (ES5)

We recommend to include the Stylesheets you need within your HTML header (between the <head> tags) and the JavaScript files directly after the starting (or before the ending) <body> tag. The plugins and language files should ALWAYS be embed AFTER the main JavaScript.

It is ALWAYS recommended to include the minified versions of the Stylesheets and JavaScript files, since it reduces the loading time for your visitors. Just for development or debugging purposes you should include the unminified files (The Stylesheets also contains a respective SourceMap pointing to the scss development files).

Global Example


<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />

        <link type="text/css" rel="stylesheet" href="dist/css/default/tail.select-default.min.css" />
        <link type="text/css" rel="stylesheet" href="dist/css/default/plugins/tail.select-{plugin}.min.css" />
    </head>
    <body>
        <script type="text/javascript" src="dist/js/tail.select.min.js"></script>
        <script type="text/javascript" src="dist/js/plugins/tail.select-{plugin}.js"></script>
        <script type="text/javascript" src="dist/js/langs/tail.select-{lang}.js"></script>

        <select>
            <option>My Option</option>
        </select>

        <script type="text/javascript">
            document.addEventListener("DOMContentLoaded", function(){
                tail.select("select", { /* Your Options */ });
            });
        </script>
    </body>
</html>

AMD Example


<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />

        <link type="text/css" rel="stylesheet" href="dist/css/default/tail.select-default.min.css" />
        <link type="text/css" rel="stylesheet" href="dist/css/default/plugins/tail.select-all.min.css" />
    </head>
    <body>
        <script type="text/javascript" src="dist/js/require.min.js"></script>
        <script type="text/javascript">
            // In this example, we assume that your JavaScript files
            //    are in the directory './dist/js/*' on your root folder
            require([
                "dist/js/tail.select",
                "dist/js/plugins/select-all",
                "dist/js/langs/all"
            ], function(select, plugin, locale) {
                plugin(select);     // Load Plugins
                locale(select);     // Load Locales

                select("select", { /* Your Options */ });
            });
        </script>

        <select>
            <option>My Option</option>
        </select>
    </body>
</html>

Use tail.select (ES6)

We recommend to include the Stylesheets you need within your HTML header (between both <head> tags) and since you're using the ECMAScript 2015 (ES6) edition, you don't need to include the JavaScript files anywhere on your HTML document, instead just import them using the import (MDN) statement like the example below.

Module Example


<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />

        <link type="text/css" rel="stylesheet" href="dist/css/default/tail.select-default.min.css" />
        <link type="text/css" rel="stylesheet" href="dist/css/default/plugins/tail.select-all.min.css" />
    </head>
    <body>
        <script type="text/javascript" src="dist/js/require.js"></script>
        <script type="text/javascript">
            // In this example, we assume that your JavaScript files
            //    are in the directory './dist/es/*' on your root folder
            import { select, options } from "./dist/es/tail.select.js";
            import { plugin as plugs } from "./dist/es/plugins/select-all.js"
            import { locale as langs } from "./dist/es/langs/all.js";

            document.addEventListener("DOMContentLoaded", function() {
                plugs(select);      // Load Plugins
                langs(select);      // Load Locales

                select("select");
            });
        </script>

        <select>
            <option>My Option</option>
        </select>
    </body>
</html>

Configure tail.select

Now you can start to configure the tail.select environment, which works the same on both versions (ES5, ES6). Just keep in mind that the used variable, where the select contructor is behind, varies depending how and which build you're including.

The global window injection, as shown on the first example, adds the global variable tail to the global window object which itself contains the main constructor on select. The AMD module, as shown on the second example, just returns the select class itself and the ES6 module, as shown on the third example, the select as well as the options classes.

In the example below, we assume that you're using the global window injection.

Constructor

The following code just shows a small example, to learn more about the select constructor check out the Select Class documentation page, the explanation about each single option can be found on the options page. 


//
//  tail.select (
//      @param      The <select> elements or the respective selector string
//      @param      The options object (the following ones are just the defaults!)
//      @param      Custom Options class, pass nothing to use the default one
//  );
//
//  You only need to pass the options you're changing the same goes for the third
//      ... parameter just pass the Options class if you're using a custom one!
//
tail.select(
    "select",                   // Selector String
    {                           // Default Options Object
        classNames: false,
        csvOutput: false,
        csvSeparator: ',',
        descriptions: true,
        deselect: false,
        disabled: false,
        grouplessName: null,
        height: 350,
        hideDisabled: false,
        hideEmpty: true,
        hideHidden: true,
        hideSelected: false,
        items: { },
        locale: 'en',
        linguisticRules: { },
        multiple: true,
        multiLimit: Infinity,
        multiSelectAll: false,
        multiSelectGroup: true,
        on: { },
        openAbove: null,
        placeholder: null,
        placeholderCount: null,
        plugins: { },
        rtl: null,
        search: false, 
        searchConfig: ['text', 'value'],
        searchFocus: true,
        searchMarked: true,
        searchMinLength: 3,
        sortItems: null,
        sortGroups: null,
        sortSearch: null,
        sourceBind: false,
        sourceHide: true,
        startOpen: false,
        stayOpen: false,
        stickyGroups: false,
        titleOverflow: 'break',
        theme: null,
        width: null
    }, 
    tail.select.options         // Default Options Class
);
